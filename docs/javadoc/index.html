Project Overview

JLib Inspector provides runtime JAR dependency tracking for Java applications through instrumentation. The API is organized into three main modules:

Package: io.github.brunoborges.jlib.agent

Java instrumentation agent that monitors JAR loading and class usage at runtime.

  • InspectorAgent - Main agent entry point
  • JarInventory - Central JAR tracking
  • ClassLoaderTrackerTransformer - Bytecode transformation
  • JLibServerClient - Server communication
  • ClasspathJarTracker - Classpath scanning

HTTP backend server for collecting and serving JAR usage data from multiple applications.

  • JLibServer - Main server class
  • AppsHandler - Application management API
  • ReportHandler - Aggregated reporting
  • HealthHandler - Health check endpoint
  • ApplicationService - Data management

Package: io.github.brunoborges.jlib.common

Shared data structures and utilities used by both agent and server components.

  • JarMetadata - JAR file metadata
  • ApplicationIdUtil - Application identification
  • JavaApplication - Application data model

Generate Full Documentation

To generate the complete interactive Javadoc with all classes, methods, and cross-references:

# Ensure you have Java 21+ and Maven 4+
java -version
./mvnw --version

# Generate aggregated Javadoc across all modules
./mvnw javadoc:aggregate

# Copy to site directory
cp -r target/site/apidocs/* site/javadoc/
